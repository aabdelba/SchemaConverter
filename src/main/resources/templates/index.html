Schema Evolver

The Goal:

To efficiently convert AVRO data from one schema to another

Did you ever need to send a POST API request to a service that constantly has different version updates?

If a QA team prefers to save expected test results in the form of JSON objects, schema changes coming in from development may require updating hundreds of expected results which often require manual effort to update. This could be a big culprit for QA scheduling delays.

In a CLI environment where applications are located in executable packages, certain applications can process hundreds of rows of JSON or AVRO records. Formatting input records when new columns or entry-relationships are introduced is extremely tedious.

The Solution, Schema Evolver

Schema evolver is a tool that formats multiple JSON or AVRO records at a time to a desired format. In order to do so, it requires two schema files, each for the old and new formats.

The intent is to format data to simply pass it through and make it compatible with the destination the data is headed. Any new columns or nested elements of data will be given default values or auto-generated values if a default is not specified.

This will be a microservice to re-arrange data to a schema of choice, with added scenarios previously not handled by AVRO

Schemas are required for this tool to work. If schemas are missing and you have classes to represent your data format, AVRO offers open-source methods for class reflection to create schemas from your old and new classes.

This tool is most suitable as an intermediary component in a CLI environment, but it is also available as a web-app and a web-service with an API.

Tool functionality:

  • Field autogenerate: initialize a new field in a record without a default value
  • Record autogenerate: initialize a new record without a default value
  • Complex data type conversion: object wrapping added
  • Complex data type conversion: array wrapping removed, create a new file for each array element
  • Complex data type conversion: enum to array
  • Complex data type conversion: array to enum
  • Primitive data type conversion to another primitive type
  • A record type is nested into a different path in the AVRO file

Support:

  • API Endpoints and methods
  • Sign-up feature
  • Support for SSH tool usage
  • Drag and drop files in the form page, instead of selecting files manually from directories
